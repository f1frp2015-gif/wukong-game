# 项目执行约定

本文件是 `wukong-game` 的持久项目记忆，适用于在本仓库工作的 Codex 及其他 AI 编程代理。

## 修改后的默认流程

- 每次完成代码、内容或配置修改后，先运行与改动相称的检查；当前最低要求是 `npm run check`。
- 检查通过后，只暂存本次任务涉及的文件，不得顺带提交用户已有的未跟踪文件或无关改动。
- 自动创建清晰的 Git 提交，并将 `main` 推送到 `origin`。本仓库内的正常修改已获得用户持续授权，可直接推送，无需每次再次询问。
- 禁止强推。若远端领先，先 `git fetch`，检查远端内容，再通过变基或正常合并保留双方改动。
- 推送后确认 Vercel 项目 `wukong-game` 的生产部署成功，并验证公开地址 `https://wukong.cab` 已包含本次版本。
- 如果 Git 集成未触发，可直接执行生产部署；生产部署属于本仓库修改后的默认交付步骤。
- 若网络、认证、权限、冲突或构建失败阻止推送/部署，必须明确报告真实状态和阻塞原因，不得把“已提交”“已触发”或命令退出码误报为上线成功。

只读检查、讨论和未产生文件变更的任务不需要提交或部署。
